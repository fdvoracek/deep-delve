{ } Deep Delve — C++-First

Build the whole game in C++ from Quest 1, with thin Blueprints that only assign data.

The working pattern

Every system is a C++ class. You then create a Blueprint subclass (BP_… / WBP_…) whose only job is to assign values to the class's UPROPERTYs — meshes, sounds, input assets, and UPrimaryDataAsset references — and to drop the actor into the level. UMG uses C++ UUserWidget bases with meta=(BindWidget) controls and WBP_ layouts. Data lives in DA_ assets.

The full quest log spells out, per quest: the C++ classes & methods created/updated, the matching Blueprints, and a UPROPERTY → value table for what you assign in each Blueprint.
