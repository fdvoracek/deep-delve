⛏️ Deep Delve

Learn Unreal Engine 5.8 by building a complete idle-mining game — pick your path.

📜 About

Deep Delve is an idle/incremental miner: click a rock to chip its HP, and when it breaks it bursts into coins. Spend them in a tech tree to sharpen and auto-swing your pickaxe, land critical hits, hire animated dwarves who mine and collect for you, upgrade your crew and veins, and descend to tougher, richer rock. You build the whole thing across a chain of 30–60 minute quests and ship a playable demo partway through.

Art comes from the Synty POLYGON Dungeon Realms pack. Each quest ends in a visible win, lists the exact Unreal tech it teaches (with doc links), and grants XP toward dwarven ranks.

🔵 Blueprints-First

The gentle on-ramp

Build everything in Unreal's visual scripting for fast, motivating wins, then pivot to C++ once the systems are real. Ship a demo before you write a line of code.

Best if: you're newer to Unreal or want to see results immediately.

🟢 C++-First

The engineer's path

Every system is a C++ class from Quest 1, with thin Blueprint subclasses that only assign data (meshes, sounds, Data Assets) and set UPROPERTY defaults. The "real Unreal" architecture, up front.

Best if: you're an experienced programmer who wants C++ from the start.

🧰 Before you start (both paths)

  • A machine that runs Unreal Engine 5.8, with ~100+ GB free disk and a modern GPU.
  • The Synty POLYGON Dungeon Realms pack — a zipped UE project you migrate into your game (Quest 1). You must own it; keep the assets out of a public repo.
  • JetBrains Rider (free for non-commercial) + a compiler toolchain (Windows: MSVC build tools; macOS: Xcode command-line tools). The C++ path needs it from Quest 1; the Blueprints path from Quest 12.
  • A GitHub account (for version control and hosting these trackers).

🎓 Further learning

Epic's free courses · Your First Hour in UE · Tom Looman (C++) · Unreal Forums. After this game: GAS, replication/multiplayer, and AI.

Roadmap: ~16–17 quests, ≈ 3–4 weeks at one sitting/day. Progress is saved per-path in your browser.
